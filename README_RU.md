# 🛡️ VPN Bot (Aiogram 3 + Marzban)

Современный асинхронный Telegram-бот для продажи и управления VPN-подписками на базе протокола **VLESS** (через панель **Marzban**). Написан на Python 3.10+, Aiogram 3 и SQLAlchemy.

!Python
!Aiogram
!Marzban

🇬🇧 English README

## ✨ Возможности

*   **🛒 Магазин подписок:** Покупка ключей на разные сроки (от 1 месяца до 1 года).
*   **🌍 Мульти-локации:** Поддержка нескольких серверов/стран (например, Швеция, Германия).
*   **👤 Профиль пользователя:** Внутренний баланс, управление подписками, статистика трафика.
*   **🎁 Пробный период:** Автоматическая выдача тестовых ключей на 24 часа.
*   **🎟️ Промокоды:** Система скидок, пополнения баланса и подарочных дней.
*   **👮 Админ-панель:** Управление пользователями, рассылка, статистика и создание промокодов.
*   **🌐 Мультиязычность:** Поддержка русского и английского языков.
*   **💳 Платежи:** Модульная структура для подключения платежных систем (CryptoBot, ЮКасса и др.).

## 🛠️ Технологии

*   **Фреймворк:** Aiogram 3.x
*   **База данных:** SQLAlchemy (Async) + PostgreSQL (via Docker) / SQLite
*   **Конфигурация:** Pydantic Settings
*   **Логирование:** Loguru
*   **Планировщик:** APScheduler

## 🚀 Установка и запуск

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/yourusername/telegram-vpn-shop.git
    cd telegram-vpn-shop
    ```

2.  **Создайте виртуальное окружение:**
    ```bash
    python -m venv venv
    source venv/bin/activate  # Linux/macOS
    # venv\Scripts\activate   # Windows
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Настройка:**
    Создайте файл `.env` в корне проекта:
    ```env
    BOT_TOKEN=ваш_токен_бота
    DB_URL=sqlite+aiosqlite:///bot.db
    ADMIN_IDS=[123456789]
    MARZBAN_HOST=https://panel.example.com
    MARZBAN_USERNAME=admin
    MARZBAN_PASSWORD=password
    ```

5.  **Запуск:**
    ```bash
    python main.py
    ```

## ⚠️ Важно: Marzban API

В текущей версии используется **Mock-сервис** (`src/services/marzban_api.py`) для тестирования интерфейса без реального сервера.
Для запуска в продакшн необходимо реализовать реальные HTTP-запросы к вашему экземпляру Marzban в этом файле.
